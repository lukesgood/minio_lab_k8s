# MinIO Kubernetes Lab - 환경 선택 가이드

## 🎯 실습 환경 선택

MinIO Kubernetes Lab은 두 가지 환경을 지원합니다. 사용자의 환경과 목적에 맞는 가이드를 선택하세요.

## 📋 환경 비교표

| 구분 | 단일 노드 환경 | 다중 노드 환경 |
|------|----------------|----------------|
| **목적** | 학습, 개발, 테스트 | 프로덕션, 고가용성 |
| **노드 수** | 1개 (control-plane) | 3개 이상 (권장: 4개+) |
| **리소스** | 4GB RAM, 2 CPU | 8GB+ RAM, 4+ CPU (노드당) |
| **스토리지** | Local Path (hostPath) | 분산 스토리지 (Ceph, GlusterFS) |
| **고가용성** | ❌ 없음 | ✅ 지원 |
| **확장성** | ❌ 제한적 | ✅ 수평 확장 |
| **성능** | 제한적 | 고성능 |
| **복잡도** | 🟢 낮음 | 🟡 높음 |
| **비용** | 🟢 낮음 | 🔴 높음 |

## 🔍 환경 자동 감지

현재 환경을 자동으로 감지하려면 다음 스크립트를 실행하세요:

```bash
./detect-environment.sh
```

## 🚀 빠른 시작

### 환경 감지 및 자동 설정

```bash
# 1. 환경 감지
./detect-environment.sh

# 2. 자동 설정 (감지된 환경에 따라)
./auto-setup.sh

# 3. 실습 시작
./run-lab.sh
```

### 수동 선택

#### 단일 노드 환경 선택
```bash
# 단일 노드 환경 설정
./setup-single-node.sh

# 단일 노드 실습 시작
./run-single-node-lab.sh
```

#### 다중 노드 환경 선택
```bash
# 다중 노드 환경 설정
./setup-multi-node.sh

# 다중 노드 실습 시작
./run-multi-node-lab.sh
```

## 🔧 환경 전환

### 단일 노드 → 다중 노드 전환

```bash
# 1. 현재 환경 정리
./cleanup-single-node.sh

# 2. 다중 노드 환경 설정
./setup-multi-node.sh

# 3. 데이터 마이그레이션 (필요시)
./migrate-data.sh
```

### 다중 노드 → 단일 노드 전환

```bash
# 1. 데이터 백업 (필요시)
./backup-data.sh

# 2. 현재 환경 정리
./cleanup-multi-node.sh

# 3. 단일 노드 환경 설정
./setup-single-node.sh
```

## 📊 성능 비교

### 단일 노드 환경 성능
```
처리량: ~100MB/s (네트워크 제한)
IOPS: ~1,000 (스토리지 제한)
동시 연결: ~100
지연시간: <1ms (로컬)
```

### 다중 노드 환경 성능
```
처리량: ~1GB/s+ (노드 수에 비례)
IOPS: ~10,000+ (분산 스토리지)
동시 연결: ~1,000+
지연시간: 1-5ms (네트워크)
```
